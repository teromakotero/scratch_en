Introduction

Welcome to learn programming with Scratch. The course consists of five parts: Statement, Loop, Condition, Variable and List. Each section has a theory part and challenges that you can use to test your skills and practice your programming skills. Challenges are tasks using the method of player learning. It is a challenging but effective way to learn programming.

The Python materials goes always side by side with this material. You can perform all tasks either in Scratch environment alone, in Python environment or both in a creative combination. The challenges are therefore completely or almost equivalent. The Scratch challenges are called Scratch_to_Python_XX and the name of the corresponding Python task would be Python_from_Scratch_XX. In other words, you can also study them simultaneously.

You will first get to know what player learning means and how the method helps you to learn programming effectively.

Idea of player learning comes from Harold Jarche.

Player Learning

Player learning does not mean that you learn by playing games, but it is a way in which a player can learn a new game in a new environment. The player tries to play a game without instructions and may fail many times. These failures are important and help players learn how to play the game gradually. Making errors develops solution models. If a player does not go further in a game, he will ask for help on the online forum. Before or after this, he may already have asked for help from a friend or colleague. Once a player has found a solution, he can share it with others. This is very close to how many programmers have learned programming and how they work every day.

Here, the principles for player learning have been divided into seven points with information on how they are used in the challenges that exist in this course.

  1. Problem without solution
    • There can be very few or no instructions. They may just show the expected end result.
    • The challenges not only measure programming skills but also the ability to solve problems.
    • The frustration and irritation you often feel when performing a difficult task is an important part of the learning process.
    • There is no single correct solution, but there may be several, one better than the other.
  2. One can find a solution without instructions
    • It is important to try different solutions and make the code continuously better.
  3. You learn from mistakes
    • To insight through trials and mistakes.
    • Looking for errors and solving them will lead to deeper learning.
  4. Making mistakes leads to solution models
    • The problem-solving abilities are developed.
    • Solution models make it easier to solve new problems.
  5. Get help from online forums
    • Programmers use online forums such as Stack Overflow to help find solutions to problems in programming.
  6. Help from friends
    • One can always ask for help from colleagues and other students.
    • Today, programming is very much a collaboration between experts.
  7. Share a solution with others
    • Many programmers share their code in GitHub and other similar sites.
    • The Scratch website at scratch.mit.edu has over 30 million distributed projects.
